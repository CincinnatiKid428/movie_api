Movie API Documentation

Objective

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

API Endpoints & Descriptions

Description URL HTTP Method Query Parameters Request Body Format Response Body Format
Returns a list of ALL movie titles /movies GET none none A JSON format response containing an array of movie titles like:

{ "movies" : [

"Stand by Me",
"The Shawshank Redemption",
"Indiana Jones: Raiders of the Lost Ark",
"Dune",
"Fight Club",
"Donnie Darko",
"Ghostbusters",
"Halloween",
"The Matrix",
"Primer"
]
}

Returns data (description, genre, director, image URL, whether it is featured or not) about a single movie by Title /movies/[Title] GET Title is the Title of the movie the user is looking for information about none A JSON format response containing information about a single movie like:

{

"Title" : "Stand by Me",
"ReleaseYear : "1986",
"Rating" : "8.1",
"Description" : "A plot summary.",
"Director" : "Rob Reiner",
"Genre" : "Drama",
"Actors" : ["Wil Wheaton", "River Phoenix", "Corey Feldman", "Keifer Sutherland"],
"ImageURL" : "https://someplace.com/stand-by-me.jpg",
"Featured" : "true"
}

Returns data about a genre (description) by name/Title (e.g., "Thriller") /genre/[Name] GET Name is the name of the genre the user is looking for information about none A text message description of the genre in the requst parameters

{

"Name" : "Drama",
"Description" : "The drama genre is characterized by..."
}

Returns data about a director (bio, birth/death year, movies) by name /director/[Name] GET Name is the name of the director to look up information none A JSON format object holding data about the director like:

{

"Name" : "Rob Reiner",
"Bio" : "A short biography."
"BirthYear" : "1947",
"DeathYear" : "",
"Movies" : ["Stand by Me", "When Harry Met Sally", "Misery", "A Few Good Men"]
}

Register a new user with username/password as user's chosen credentials /users POST none A JSON object holding the desired information to register a new account like:

{

"Username" : "Saffron",
"Password" : "meow1234!",
"Email" : "risotto@email.com",
"Birthdate" : "2022/10/31"
}

A JSON object holding the new account information or error message if creation failed:

{

"_id": "672646c255a517151e82293c",
"Username" : "Saffron",
"Password" : "(Hashed Password)",
"Email" : "risotto@email.com",
"Birthdate" : "ISODate('2022-10-31T04:00:00.000Z')",
"FavoriteMovies" : []
}

Updates an exisiting user's account information. /users/[Username] PUT Username is the account Username to receive updated information in request body. A JSON object with account information to be updated.

{

"Username" : "Fred09",
"Password" : "abc555!",
"Email" : "fred09@email.com",
"Birthdate" : "1978/05/26"
}

A JSON object holding the updated account information or message of failure to update:

{

"_id": "672646c255a517151e82293c",
"Username" : "Fred09",
"Password" : "(Hashed Password)",
"Email" : "fred09@email.com",
"Birthdate" : "ISODate('1978-05-26T04:00:00.000Z')",
"FavoriteMovies" : []
}

Adds a movie by MovieID to a favorites list /movies/favorites/[MovieID] POST MovieID is the _id field of the movie to be added to the user's list none A JSON response with updated User favorite movies or error due to failure:

{

"_id": "672646c255a517151e82293c",
"Username" : "Saffron",
"Password" : "(Hashed Password)",
"Email" : "risotto@email.com",
"Birthdate" : "ISODate('2022-10-31T04:00:00.000Z')",
"FavoriteMovies" : ["671ebe39577ed8cab386b022"]
}

Removes a movie by MovieID from a favorites list /movies/favorites/[MovieID] DELETE MovieID is the _id field of the movie to be removed the user's list none A JSON response with updated User favorite movies or error due to failure:

{

"_id": "672646c255a517151e82293c",
"Username" : "Saffron",
"Password" : "(Hashed Password)",
"Email" : "risotto@email.com",
"Birthdate" : "ISODate('2022-10-31T04:00:00.000Z')",
"FavoriteMovies" : []
}

Deregisters a user given valid credentials /users DELETE none

{

"Username" : "Saffron"
}

A text response confirming success/failure of deregister operation
