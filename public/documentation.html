Movie API Documentation

Objective

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

API Endpoints & Descriptions

Description URL HTTP Method Query Parameters Request Body Format Response Body Format
Returns a list of ALL movies /movies GET none none A JSON format response containing an array of movies like:

{
"movies" : [

{"title" : "Stand by Me"},
{"title" : "The Shawshank Redemption"},
{"title" : "Indiana Jones: Raiders of the Lost Ark"},
{"title" : "Dune"},
{"title" : "Fight Club"},
{"title" : "Donnie Darko"},
{"title" : "Ghostbusters"},
{"title" : "Halloween"},
{"title" : "The Matrix"},
{"title" : "Primer"}
]
}

Returns data (description, genre, director, image URL, whether it is featured or not) about a single movie by title /movies/[title] GET
  • title is the title of the movie the user is looking for information about
 none A JSON format response containing information about a single movie like:

{
"movie" : [

{"title" : "Stand by Me"},
{"year" : "1986"},
{"genre" : "Adventure, Comedy, Drama"},
{"director" : "Rob Reiner"},
{"imgURL" : "https://someplace.com/stand-by-me.jpg"},
{"featured" : "true"},
]

}

Returns data about a genre (description) by name/title (e.g., "Thriller") /genre/[name] GET
  • name is the name of the genre the user is looking for information about
 none A text message description of the genre in the requst parameters
Returns data about a director (bio, birth year, death year) by name /director/[name] GET
  • name is the name of the director to look up information
 none A JSON format object holding data about the director like:

{
"director" : [

{"name" : "Rob Reiner"},
{"birthYear" : "1947"},
{"deathYear" : ""},
{"bio" : "A short biography."}
]

}

Register a new user with username/password as user's chosen credentials /users/[username]/[password] POST
  • username is the user's desired login
  • password is the user's desired password
 none A text message if the user account was created or not
Updates an exisiting user's username given valid credentials /users/newUsername/[username]/[password]/[newUsername] PUT
  • username is the user's login
  • password is the user's password
  • newUsername is the user's desired login
 none A text message notifying of success/failure with updating username
Adds a movie by title to a favorites list and return text response /users/favorites/[username]/[password]/[title] POST
  • username is the user's login
  • password is the user's password
  • title is the name of the movie to be added to the user's list
 none A text message indicating if adding the movie to list was a success/failure
Removes a movie by title from a favorites list and return text response /users/favorites/[username]/[password]/[title] DELETE
  • username is the user's login
  • password is the user's password
  • title is the name of the movie to be removed the user's list
 none A text response indicating success/failure of removing the favorite movie from list
Deregisters a user given valid credentials /users/[username]/[password] DELETE
  • username is the user's login
  • password is the user's password
 none A text response confirming success/failure of deregister operation
